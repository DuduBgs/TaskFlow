# Plano de Trabalho

## Objetivo
- 
O sistema tem como objetivo resolver a dificuldade de organização e acompanhamento de tarefas em projetos acadêmicos ou profissionais, centralizando as informações em uma única plataforma.

Muitas equipes enfrentam problemas como falta de clareza sobre responsabilidades, prazos e andamento das atividades.
O TaskFlow busca facilitar essa gestão por meio de um ambiente simples e intuitivo.
## Escopo
- 
O sistema permitirá que usuários criem projetos e gerenciem tarefas relacionadas a eles.

De forma geral, a aplicação irá:

cadastrar usuários
criar projetos
cadastrar tarefas
atribuir tarefas a membros da equipe
acompanhar status e prazos
editar e remover tarefas
visualizar andamento do projeto

## Observações
- Tecnologias que a equipe pretende utilizar
- Front-end: React
- Back-end: Node.js com Express
- Banco de dados: PostgreSQL
- Containerização: Docker
- Orquestração: Docker Compose
- Versionamento: GitHub + GitFlow
- CI/CD: Jenkins
- Qualidade de código: SonarQube

- CRUD principal

- O CRUD principal será o gerenciamento de tarefas.

- A aplicação permitirá:

- Create: cadastrar nova tarefa
- Read: listar tarefas cadastradas
- Update: editar título, descrição, prazo e status
- Delete: remover tarefas

Esse será o módulo principal do sistema, sendo o foco central do projeto.
